Quick Start Guide

This Quick Start guide is designed to get you up and running with PageIntel immediately after installation.

PageIntel is a powerful Chrome extension that allows you to extract and analyze web content based on your specific needs. This quick start guide will help you get up and running with PageIntel in no time.

Step 1: Obtain an API Key

To use PageIntel, you need to obtain an API key from OpenAI. Follow these steps:

  1. Visit the OpenAI website and sign up for an account.
  2. Navigate to the API section and generate a new API key.
  3. Copy the API key to your clipboard.

Step 2: Configure PageIntel

Open the PageIntel extension and follow these steps to configure it:

  1. Click on the PageIntel icon in your browser's toolbar to open the extension.
  2. In the settings area, paste your OpenAI API key into the designated field.
  3. Customize other settings according to your preferences, such as the AI model and display options.
  4. Click the "Save" button to apply your changes.

Step 3: Create Custom Tasks

PageIntel allows you to create custom tasks to extract and analyze data from web pages. Here's how:

  1. In the PageIntel extension, navigate to the "Custom Tasks" section.
  2. Click the "Add New Task" button to create a new custom task.
  3. Provide a name and description for your task.
  4. Define the task using the available data tags and custom logic. For example: {page-title} - {page-url}
  5. Save your custom task.

Step 4: Use Data Tags

Data tags are placeholders that you can use within your custom tasks to refer to specific elements or data points on a web page. PageIntel provides a set of predefined data tags, such as {page-title}, {page-url}, and {selection}.

You can also create your own custom data tags by defining CSS selectors or JSON paths in the "Data Tags" section of the extension.

Step 5: Run Custom Tasks

To run a custom task, follow these steps:

  1. Navigate to a web page where you want to extract or analyze data.
  2. Open the PageIntel extension.
  3. Select the desired custom task from the dropdown menu.
  4. Click the "Run Task" button to execute the task.
  5. View the results in the extension's output area.

Step 6: Explore Advanced Features

PageIntel offers several advanced features to enhance your experience:

  • Customize the AI model for better results.
  • Enable or disable the display of results in a popup window.
  • Use the keyboard shortcut Ctrl+Shift+P to quickly open PageIntel.
  • Access the extension's options page for more granular control over settings.

That's it! You're now ready to use PageIntel to supercharge your web browsing experience. If you have any questions or need further assistance, please refer to our documentation or contact our support team.

Happy browsing with PageIntel!

Main Section and Main Menu

The main section of the PageIntel extension provides a convenient way for you to interact with predefined tasks and ask impromptu questions. This section is designed to streamline your workflow and make it easy for you to execute tasks and get answers to your queries.

Running Predefined Tasks

PageIntel comes with a set of predefined tasks that you can execute with just a click. These tasks are displayed in the main section as a list of task items. To run a task, simply locate the desired task in the list and click on the "play" icon (▶) associated with it. PageIntel will then execute the task and display the results in the designated output area.

Asking Impromptu Questions

In addition to predefined tasks, PageIntel allows you to ask impromptu questions directly from the main section. At the top of the task list, there is a dedicated input field where you can type your questions. Simply enter your question into this field and press Enter or click on the "play" icon (▶) to execute the query. PageIntel will process the question and provide a response based on its AI capabilities.

Saving Impromptu Questions as Tasks

If you find yourself frequently asking similar questions or if you want to save an impromptu question for future use, PageIntel provides the ability for you to save these questions as predefined tasks. After entering a question and receiving a satisfactory response, click on the "save" icon (💾) next to the question input field. You will be prompted to enter a name for the task. Once saved, the question will appear in the task list as a new predefined task.

To further refine the saved task, you can go to the settings panel by clicking on the gear icon (⚙) in the main menu. In the settings panel, locate the newly saved task and click on it to expand its details. Here, you can add a description for the task, specifying its purpose or any additional information. Additionally, you can define the applicable URLs where the task should be available. This allows you to limit the task's visibility to specific web pages or domains.

Task Filtering

As you accumulate more predefined tasks, the task list in the main section can become lengthy. To help you quickly find the desired task, PageIntel provides a task filtering feature. In the main menu, located at the top of the main section, you will find a search icon (🔍). Clicking on this icon will reveal an input field where you can enter keywords or phrases to filter the task list. As you type, the task list will dynamically update to show only the tasks that match your search query. To clear the filter and display all tasks again, click on the clear icon (❌) next to the filter input field.

The main section and main menu of PageIntel provide a user-friendly interface for you to execute predefined tasks, ask impromptu questions, and manage your saved tasks. With the ability to save questions as tasks, refine them in the settings panel, and filter the task list, PageIntel empowers you to streamline your workflow and quickly access the information you need.

AI Tasks: The Core of PageIntel

AI Tasks are the foundational element of PageIntel, centralizing the extension's capabilities to automate and enhance user interactions with web content using artificial intelligence.

AI Tasks are the foundational element of PageIntel, centralizing the extension's capabilities to automate and enhance user interactions with web content using artificial intelligence. These tasks are designed to simplify complex processes, from data extraction to content analysis, by intelligently processing web page data.

How AI Tasks Work

The workflow of AI Tasks in PageIntel is a systematic process that integrates data extraction with AI processing:

  1. Task Definition: Users define AI Tasks using a specific prompt structure that may include placeholders for data tags.
  2. Data Tag Execution: When an AI Task is triggered, PageIntel first identifies the data tags embedded within the task prompt. These tags are used to extract specific data elements from the current web page.
  3. Data Replacement: Extracted data replaces the corresponding data tags in the AI Task prompt, transforming it into a complete query ready for processing.
  4. AI Processing: The fully formed query is sent to an AI model, which processes the information and generates a response based on the input.
  5. Response Display: The AI's response is then displayed back to the user, providing insights, summaries, or any other requested analysis.

Writing Effective AI Task Prompts

Writing effective prompts for AI Tasks is crucial for achieving accurate and relevant AI responses. The prompts should be clear and direct, with properly placed data tags to ensure precise data extraction. Here are some tips and examples for prompt writing:

  • Be Specific: Clearly state what the AI needs to do with the data once extracted. The more specific the prompt, the more accurate the AI's response.
  • Use Data Tags Wisely: Place data tags where the extracted data will be most meaningful for the AI's processing.
  • Context Matters: Provide enough context in the prompt to guide the AI's response in the right direction.

Example prompts include:

  • Price Comparison Task: "Compare the current price of {product-name} at {product-price} with historical prices and advise if it's a good time to buy."
  • Content Summary Task: "Summarize the key points from the following article: {page-full-content}."

URL Filters and Wildcards in AI Tasks

URL filters in PageIntel define the scope of web pages where AI Tasks will be activated, allowing for targeted application of tasks to relevant content. The flexibility of URL filters is crucial for ensuring that tasks operate efficiently and only when necessary.

  • Blank Filters: If no URL filter is specified, the AI Task will be considered applicable to all pages. It is important that such a task is general enough to be relevant across different webpages.
  • Single and Multiple Filters: Users can specify single or multiple URL filters to narrow down the task's applicability. For example, a price monitoring AI Task might be set to trigger on URLs like *amazon.com/products*, *ebay.com/offers*.
  • Use of Wildcards: Wildcards (*) allow flexibility in matching URL patterns, enabling the task to be triggered on broader URL patterns such as *.newswebsite.com/* which would cover all subdomains of newswebsite.com.
  • Cross-Domain Filters: For tasks that require data from multiple sources, URL filters can span different domains, such as *cnn.com/news*, *bbc.com/news* to compare news coverage on similar topics across different news websites.

By using URL filters effectively, users can ensure that AI Tasks are triggered only under appropriate conditions, maximizing the extension's utility and performance across varied web environments. This selective activation helps conserve resources and ensures that the output from AI Tasks is as relevant and useful as possible.

Data Tags in PageIntel

Data tags allow you to define and extract specific data elements from web pages automatically, enhancing your ability to automate and streamline data collection tasks.

PageIntel's data tags are a core feature that allow users to define and extract specific data elements from web pages automatically. This feature enhances the ability to automate and streamline data collection tasks tailored to your specific needs.

Overview of Data Tags

Data tags in PageIntel act as identifiers for specific data within a web page. These can be anything from simple elements like page titles or URLs to more complex data such as prices or specific text blocks within an article.

Creating Data Tags

  1. Access the Data Tags Panel: Open the PageIntel settings and navigate to the Data Tags section.
  2. Add a New Tag: Click the "Add New Tag" button to start defining a new data tag.
  3. Define the Tag:
    • Tag Name: Enter a unique name for the data tag.
    • Tag Description: Provide a brief description of what the tag is used for.
    • CSS Selector/JSON: Enter the CSS selector or JSON path that defines where to extract the data from the page.
  4. Save the Tag: Click the "Save Tag" button to store the tag in your extension settings.

Using Data Tags

Once a tag is created, it can be utilized in various tasks throughout the extension. For example:

  • Automated Scraping: Automatically extract and save the data specified by the tag when visiting a web page.
  • Data Analysis: Use the tag to gather specific types of data for analysis, such as extracting all product prices from an e-commerce site.
  • Task Automation: Incorporate tags into custom tasks to process or monitor specific information on web pages.

Testing and Validation

Before fully implementing a new data tag, it is important to test its functionality:

  1. Select the Tag: In the Data Tags section, select the tag you wish to test.
  2. Test the Tag: Click the "Test Tag" button to execute the tag on the current page.
  3. Review the Output: Ensure that the tag correctly captures the data as expected.

Advanced Data Tagging: CSS Selectors and JSON Paths

CSS Selectors

CSS selectors are patterns used to select the elements you want to style in a webpage. In PageIntel, you use CSS selectors to define which elements to extract data from. Examples include simple selectors like div, #header, or .price, and combinator selectors like div > p which selects all <p> elements that are direct children of a <div>.

JSON Paths for Advanced Tagging

For more complex structures such as iframes or shadow DOMs, JSON paths are used. These provide a structured way to navigate through nested elements or across different document contexts like iframes or shadow DOMs.

{
                    "selectors": [
                        {
                            "iframe_path": ["iframe#main", "iframe#content"],
                            "shadow_dom_path": ["div#shadowHost", "shadowRoot"],
                            "element_selector": "p.price"
                        }
                    ]
                }

This JSON path configuration allows navigation into nested iframes, through shadow DOMs, and directly to the elements of interest, such as p.price.

Use Cases for JSON Paths

Example use case includes extracting product prices from an e-commerce platform where product details are loaded within an iframe and prices are displayed within shadow DOMs:

{
                    "selectors": [
                        {
                            "iframe_path": ["iframe#productFrame"],
                            "shadow_dom_path": ["div#productDetails", "shadowRoot"],
                            "element_selector": "span.price"
                        }
                    ]
                }

Testing and Validation

Set up the tag with the appropriate JSON path or CSS selector, test it on a relevant page, and check the results to ensure accurate data extraction.

By effectively using data tags, PageIntel users can tailor the extension to meet their precise data collection and monitoring requirements, making web interactions more efficient and productive.

Built-In Data Tags in PageIntel

PageIntel includes a set of built-in data tags that provide essential functionality for capturing common types of data from web pages without the need for custom configuration. These tags are pre-defined within the extension and are ready to use immediately for various data extraction tasks.

Overview of Built-In Tags

Built-in tags in PageIntel are designed to simplify common data extraction tasks by providing predefined selectors and handlers. These tags target standard web page elements and attributes, such as page titles, URLs, and more.

List of Built-In Data Tags

Here are some of the most commonly used built-in data tags provided by PageIntel:

  • page-title: Captures the title of the current web page.
    • Handler: Returns the content of the <title> tag from the HTML document.
  • page-url: Captures the URL of the current web page.
    • Handler: Returns the full URL from the browser's address bar.
  • page-input-data: Extracts data from input fields within the web page.
    • Handler: Collects values from all input elements like text boxes, checkboxes, and radio buttons.
    • Output: Returns a JSON string containing details such as input names, types, values, and associated labels.
  • page-full-content: Captures all readable text from the web page, excluding scripts, styles, and media elements.
    • Handler: Iteratively collects text content from visible elements, navigating through nested structures including iframes and shadow DOMs.

Using Built-In Data Tags

To utilize built-in data tags in PageIntel:

  1. Access Data Tags Section: Open the settings or tag management panel in PageIntel.
  2. Select a Built-In Tag: Choose a tag from the list of built-in tags. These are usually marked with a special notation or different color to distinguish them from custom tags.
  3. Apply the Tag: Use the tag in tasks such as data scraping, monitoring, or analysis. Built-in tags can be integrated into workflows just like custom tags.

Advantages of Built-In Data Tags

  • Ease of Use: No setup or configuration is required to start using built-in tags.
  • Reliability: These tags are tested extensively to ensure they perform well across a wide range of websites.
  • Efficiency: Speed up the development of data collection tasks by using pre-defined, commonly needed data selectors.

Example Use Case

Imagine you are conducting market research and need to extract the titles and URLs from a list of blog posts on a news site. By using the page-title and page-url built-in tags, you can quickly set up a task in PageIntel to scrape this information from each page visited, storing the results for later analysis or reporting.

Built-in data tags are a powerful feature of PageIntel, enabling both novice and advanced users to perform complex data extraction tasks with minimal effort. Whether you are tracking price changes, analyzing article content, or gathering metadata from web pages, these tags provide a solid foundation for your data extraction needs.

Advanced Settings

Explore the advanced settings of PageIntel, including Import/Export facilities, AI model selection, and more.

Import/Export Facilities

PageIntel provides robust options for importing and exporting settings and configurations, allowing users to easily backup their settings or transfer them to another installation.

Exporting Settings

  1. Navigate to the Settings Panel: Access the settings panel from the PageIntel interface.
  2. Export Settings: Click on the "Export Settings" button. This will generate a JSON file containing all your customized settings, tasks, and data tags.
  3. Download the File: The JSON file will be automatically downloaded to your default downloads folder.

The exported settings include all custom configurations such as API keys, model selections, task definitions, and data tag setups.

Importing Settings

  1. Open the Settings Panel: Access this panel through the main PageIntel interface.
  2. Choose Import Source:
    • File: Select "Import from File" if you have a JSON settings file. Click the file input area to select and upload the file from your local system.
    • URL: Select "Import from URL" if the settings are hosted online. Enter the URL where the JSON file is located.
  3. Select Import Mode: Choose whether to "Merge with existing settings" or "Overwrite existing settings". Merging will combine the imported settings with the existing ones, while overwriting will replace all current settings with those imported.
  4. Execute the Import: Click the "Import" button to apply the imported settings.

The import process is designed to be straightforward and user-friendly, with clear options for managing how incoming settings are integrated with existing configurations.

By utilizing these features, PageIntel users can ensure that their configurations are preserved across different installations or easily shared with colleagues, enhancing the utility and flexibility of their setup.

Additional Features

  • Password Protection: Set a password for accessing the settings panel to prevent unauthorized changes.
  • API Model Selection: Choose between different AI models (e.g., GPT-3.5-Turbo or GPT-4-Turbo) for processing tasks.

AI Key and AI URL Configuration

Learn how to configure the AI Key and AI URL to enhance the flexibility and security of AI tasks in PageIntel.

The configuration of the AI Key and AI URL in PageIntel is a crucial aspect that enhances the flexibility and security of how AI tasks are handled within the extension. This setup not only allows for seamless integration with enterprise systems but also supports advanced customizations and secure data handling.

AI Key Configuration

The AI Key is a critical component in PageIntel, enabling the extension to authenticate and interact with AI services. This key is typically used to authorize API calls made to AI services like OpenAI or any other large language models (LLMs) that the extension might communicate with.

  • Security: The AI Key is securely stored and managed within the user's PageIntel setup, ensuring that sensitive credentials are kept safe from unauthorized access.
  • Ease of Use: Users can easily enter and update their AI Key through the PageIntel settings panel, allowing for straightforward configuration and maintenance.

AI URL Configuration

The AI URL configuration plays a vital role in how PageIntel interacts with different AI services. By allowing the AI URL to be configurable, PageIntel can support advanced setups, such as routing API calls through an API gateway.

  • Enterprise Integration: In an enterprise setup, the API call can be made to an API gateway instead of directly to an AI service. This gateway can reformat the API calls, potentially routing them to different AI services based on the needs and policies of the enterprise.
  • User Authentication: The API gateway can handle user authentication via Single Sign-On (SSO) or other authentication methods. This ensures that AI interactions comply with the enterprise's security standards and that user credentials are managed securely.
  • Key Management: The API gateway can manage the AI Key, meaning that the actual AI service key does not need to be stored within the user's PageIntel setup. This adds an extra layer of security, as the sensitive AI Key is managed and protected by the gateway, reducing the risk of exposure.
  • Flexibility and Customization: Organizations can customize the AI URL to point to different endpoints depending on specific requirements, such as load balancing, failover strategies, or specific data handling rules that comply with local regulations.

The default entry for the AI URL is the OpenAI API URL, as this is likely the most common use case. The API call is made in the format required by OpenAI. If the target LLM uses a different format, the message can be translated to their format by the API gateway.

Implementation Considerations

When configuring the AI URL and AI Key in PageIntel, consider the following:

  • Compliance and Security: Ensure that the configuration adheres to your organization's IT security policies and compliance requirements.
  • Testing and Validation: Before fully implementing these settings in a production environment, conduct thorough testing to verify that the AI tasks perform as expected and that all security measures are robust.
  • Documentation and Support: Maintain detailed documentation of the configuration settings and ensure that technical support is available to address potential issues during setup or operation.

By leveraging the flexibility of configurable AI URLs and secure AI Key management, PageIntel can effectively meet the diverse needs of individual users and organizations, enhancing the security and efficiency of AI-driven tasks.
